# AI 模型架构

平台采用**大模型 + 本地小模型双模式**：现场默认离线运行本地 ML/DL 小模型（纯 CPU，无网络依赖）；接入内网大模型网关后升级为 7B 多模态大模型分析。两种模式接口完全一致（`ml_models.py` 与 `llm_models.py` 同构），通过环境变量一键切换、故障时自动回退。

| 模式 | 配置 | 适用场景 |
|---|---|---|
| 本地小模型（默认） | `MODEL_TYPE=local` | 纯离线、CPU 服务器、无网关 |
| 7B 多模态大模型 | `MODEL_TYPE=llm` + `QWEN_MODEL=qwen2-vl-7b-instruct` | 内网大模型网关可达 |

## 1. 7B 多模态大模型（国网光明电力大模型）

### "7B" 具体是什么模型

- **基座模型**：**Qwen2-VL-7B-Instruct**（阿里通义千问 2 视觉语言模型，7B 指令微调版）。"7B" 即约 **70 亿参数**的模型规模。
- **与"国网光明电力大模型"的关系**："国网光明电力大模型"是国网侧**内网大模型网关服务**的名称；该网关后端部署的具体模型即 Qwen2-VL-7B-Instruct。平台不直接持有模型权重，而是调用网关暴露的推理服务。
- **选型理由**：
  1. **多模态**：自带视觉理解能力，可直接分析监控截图、巡检记录照片，这是纯文本模型做不到的；
  2. **可内网部署**：7B 规模在单张主流推理卡（如 A10/A800 级）上即可服务，满足电网内网纯离线、数据不出网的合规要求；
  3. **中文能力强**：通义千问系列中文运维语料理解与中文报告生成质量好，开源协议允许商用。
- **接入方式**：通过网关的 **OpenAI 兼容接口**（`/chat/completions`）调用，网关地址与 API Key 全部由环境变量注入（`QWEN_API_BASE` / `QWEN_API_KEY` / `QWEN_MODEL=qwen2-vl-7b-instruct`），不硬编码。
- **多模态能力**（`models/llm_models.py: LLMClient.chat_with_image`）：支持图文混合输入，用于监控截图分析、巡检记录照片识别，结果直接写入运维报告。
- **承担的任务**：
  | 任务 | 说明 |
  |---|---|
  | 智能问答 | 多轮上下文连贯的运维对话（追问闸门 + 历史注入），可结合实时指标回答 |
  | RAG 知识检索对话 | 检索结构化知识库（故障模式/SOP/历史案例）后注入上下文回答 |
  | 故障分类归因 | `LLMFaultClassifier` 把实时指标交给大模型输出结构化故障判定（JSON），失败自动回退规则分类 |
  | NLP 报告生成 | 按系统深度分析报告、故障处置报告的中文长文生成 |
  | 图文分析 | 监控截图/巡检照片的多模态理解 |
- **工程保障**：5 分钟响应缓存降低网关压力；超时/异常自动回退本地模型，保证功能永不因网关故障中断。

## 2. 本地 ML 模型（`models/ml_models.py`，纯 CPU）

| 模型 | 方法 | 用途 |
|---|---|---|
| FaultClassifier | 规则 + 统计（20+ 故障模式条件库） | 实时指标 → 故障类型/严重级别（CPU 过载、内存不足、IO 瓶颈等） |
| AnomalyDetector | 阈值 + 统计（均值/方差偏离） | 指标异常检测，供告警与自愈守护触发 |
| 文本分类器 | TF-IDF | 运维文本/日志分类与知识检索打分 |
| 轻量语义模型 | 2 层 Transformer（hidden 128, 4 heads, max_len 256） | 离线语义理解，支撑无网关时的问答兜底 |

## 3. 本地 DL 模型（`models/dl_models.py`，PyTorch CPU）

| 模型 | 结构 | 用途 |
|---|---|---|
| SimpleNN | MLP 10→64→32→5（ReLU + Dropout + BatchNorm） | 10 维系统指标 → 5 类故障（正常/CPU/内存/IO/网络）概率分布 |
| LSTMAnalyzer | 2 层 LSTM，hidden 64 | 指标时序趋势分析 |
| MiniTransformerEncoder | 小型 Transformer 编码器（位置编码 + 自建词表） | 运维文本分析 |
| AutoEncoder | 编码 10→4 重建 | 重建误差异常检测（与统计检测互补） |
| AttentionClassifier | 注意力分类器 | 故障分类 + 注意力权重输出**可解释**的指标归因 |

统一由 `DLModelManager` 管理（权重初始化可复现、checkpoint 保存/加载、综合分析入口）。

## 4. 智能修复决策模型（`models/repair_decision_model.py`）

混合决策器 `HybridRepairDecisionMaker` = **DL 端到端网络 + ML 决策树** 双通道：

```
故障检测结果 (指标 + 分类 + 根因)
        │ encode_fault_data → 20 维特征
        ├──► DL 通道 IntelligentRepairModel
        │      FaultFeatureExtractor (20→32 特征提取)
        │      → RepairStrategyClassifier (32→10 类修复策略)
        │      → RepairSuccessPredictor (策略嵌入 + 特征 → 成功率预测)
        ├──► ML 通道 MLDecisionTree（可解释规则决策树）
        └──► 混合仲裁 → 最终策略 + 预估成功率 + 决策依据
                │
                ▼
        Agent /repair 执行对应 playbook（10 类修复 action）
```

修复结果回流历史库，用于策略自适应调整与 KPI 看板（成功率、MTTR）统计。

## 5. 趋势预测（`services/trend_predictor.py`）

后台持续采样各服务器指标，对滑动窗口做**线性回归外推**，输出未来 24 小时 CPU/内存/磁盘趋势曲线与风险评分（到达阈值的预计时间），支撑"状态推演与预测"和巡检排程的风险侧重。

## 6. 模型分工总览

```
                         ┌────────────────────────────┐
   实时指标/巡检数据 ────►│ ML: 异常检测 + 故障分类      │──► 告警聚合 → 根因规则
                         │ DL: 分类/时序/异常/可解释    │──► 修复决策(混合) → 自愈执行
                         │ 线性回归: 24h 趋势 + 风险    │──► 巡检排程
                         └────────────────────────────┘
                                      │ 汇总上下文
                                      ▼
                         ┌────────────────────────────┐
   知识库(RAG) + 图片 ───►│ 7B 多模态大模型（内网网关）  │──► 问答 / 报告 / 图文分析
                         └────────────────────────────┘
```

设计取舍：**小模型管"快"**（秒级检测、10 秒自愈闭环，离线可用），**大模型管"深"**（归因解释、报告生成、知识对话、多模态），互为兜底，保证平台在纯离线与有网关两种现场条件下都完整可用。
