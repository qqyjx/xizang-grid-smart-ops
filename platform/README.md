# 西藏电网智能运维平台 - 7B多模态版 v5.32

## 概述

基于内网 **7B多模态大模型** 的智能运维平台。支持本地小模型和7B大模型一键切换，所有功能与 Mini 版完全一致，额外新增图片分析能力。

**部署架构**：

```
监控服务器                          被监控服务器
┌──────────────────────┐           ┌─────────────────┐
│  xizang-offline-7B   │◄──HTTP───►│  xizang-agent-7B │ × N台
│  端口: 5001           │  (8089)   │  端口: 8089       │
│  Web界面 + AI分析     │           │  指标采集 + 修复   │
└──────────────────────┘           └─────────────────┘
         │
         ▼
   内网大模型网关
   <内网IP>:80
```

## 快速部署

```bash
# 解压
tar -xzf xizang-offline-7B-v5.25.tar.gz
cd xizang-offline-7B

# 一键部署（API已预配置，无需设置任何环境变量）
chmod +x deploy.sh && ./deploy.sh

# 验证
curl http://localhost:5001/api/health
```

部署脚本会自动：安装 Python（Miniconda）→ 安装依赖（离线 wheel）→ 启动服务。全程无需联网。

## 功能清单

| 功能模块 | 说明 |
|---------|------|
| **智能问答** | 文本对话 + 图片上传分析（7B多模态） |
| **运行监控** | CPU/内存/磁盘/IO/连接数实时图表 |
| **自动修复** | 故障检测 + 一键修复 + 脚本执行 |
| **生成报告** | 综合运维报告 + 真实服务器报告 |
| **知识库** | 故障模式库(20) + 处置方案库(20) + 历史案例库(16) |
| **模型切换** | 本地小模型(ML+DL) ↔ 7B大模型，前端一键切换 |

### 与 Mini 版的关系

7B 版 = Mini 版全部功能 + 多模态图片分析。**没有任何功能删减**，只有新增。

## 模型切换

点击左下角齿轮图标 → 模型配置：

| 模式 | 说明 | 适用场景 |
|------|------|---------|
| **本地小模型** | ML+DL，纯CPU，无需网络 | 离线环境、基础分析 |
| **7B大模型** | 内网多模态API，支持图文分析 | 需要深度分析、图片识别 |

切换后点"保存配置"即生效。可随时点"测试连接"验证7B大模型连通性。

> 切换模型后需要**重新生成**报告，之前已生成的报告内容不会自动改变。

## 智能问答 — 支持的输入

### 文本输入
直接在对话框输入问题，例如：
- "服务器CPU使用率持续90%怎么处理"
- "分析最近的系统状态"
- "帮我检查所有服务器的磁盘使用情况"

### 图片上传（7B大模型模式）
点击对话框左侧的图片按钮或拖拽上传，支持的图片类型：
- **监控大屏截图** — Zabbix/Grafana/云监控等监控系统的告警页面
- **终端截图** — top、htop、iostat、df -h 等命令输出的截图
- **设备巡检照片** — 机房设备、指示灯状态、线缆连接等
- **日志截图** — 系统日志、应用日志的关键错误信息

支持格式：PNG、JPEG、GIF、WebP，建议分辨率 480×480 ~ 2560×2560。

### 文档上传
点击对话框的文件按钮，支持的文档类型：
- `.log` — 系统日志、应用日志
- `.txt` — 命令输出、状态信息
- `.json` — 监控数据、配置文件
- `.csv` — 指标数据、统计报表

## 运行监控

### 时间范围选择
监控页面右上角可选择：最近5分钟 / 15分钟 / 30分钟 / 1小时。

> Agent 刚部署时历史数据较少，切换时间范围效果不明显。运行一段时间后数据会自动积累。

### 监控数据存储
Agent 的历史监控数据存储在**平台端内存中**（保留最近30分钟），非持久化存储。服务重启后历史数据会清空，但 Agent 会立即开始重新采集。

### 刷新
点击"刷新"按钮会重新拉取所有服务器的最新指标。如果数据没有明显变化，界面看起来可能和刷新前一样——这是正常的。

## 自动修复

### 脚本编辑
在自动修复页面，点击齿轮图标（检修脚本配置），可以编辑以下脚本：
- 健康检查脚本
- 备份脚本
- 清理脚本
- 服务重启脚本
- 重启服务器

每个脚本支持：自定义命令、设置超时时间、是否需要 sudo 权限。

## 生成报告

两种报告类型：

| 类型 | 说明 | 下载文件名 |
|------|------|-----------|
| 综合运维报告 | 所有服务器状态、故障分析、修复记录 | `综合运维报告_日期.md` |
| 真实服务器报告 | 专注 Agent 部署的真实服务器监控 | `真实服务器报告_日期.md` |

报告中的"分析引擎"会根据当前选择的模型动态显示（7B多模态 / ML+DL）。

## 知识库

左侧菜单第5项"知识库"，包含：

- **故障模式库**（20种）— CPU过高、内存泄漏、磁盘满、网络异常、IO瓶颈、僵尸进程、服务崩溃、高连接数、Swap频繁、时间偏移、安全审计异常、证书过期、DNS故障、NTP同步失败、文件系统只读、OOM Killer、端口冲突、SSH超限、高负载、内核参数异常
- **处置方案库**（20种）— 每种故障对应的处置步骤、常用命令、预防措施
- **历史案例库**（16个）— 真实故障案例，含根因分析、解决过程、预防建议

支持关键词搜索，输入故障特征即可匹配相关知识。

## 常用命令

```bash
./deploy.sh              # 部署/重启
./stop.sh                # 停止服务
tail -f logs/app.log     # 查看实时日志
./diagnose.sh            # 一键诊断（功能异常时生成报告给技术支持）
```

## 功能异常时怎么办

**两样东西一起发给技术支持**：

1. **出问题的页面截图**（手机拍屏或系统截图工具都行）
2. **诊断报告** — 在服务器上运行：

   ```bash
   cd xizang-offline-7B && ./diagnose.sh
   # 生成 diagnose_report_<时间戳>.txt
   ```

   或点页面右上角"**诊断报告**"按钮下载 JSON。

> 诊断报告自动包含：系统环境 / Python 环境 / 配置 / LLM 连通性 / 服务状态 / 知识库数据 / Agent 连通性 / 应用日志尾部 / 分组字段分布 — **API Key 和 Token 已脱敏**，可放心发送。
