自动检修怎么用？（点开看 3 步）
用途：对选定服务器主动跑一次"体检"——执行健康检查/备份/清理等预设脚本，或重启服务/整机，用于日常巡检与故障前的主动排查（区别于"自动修复"是发现故障后回收处置）。
三步操作：
① 选「检修服务器」（系统多时先用上方"按系统筛选"收窄）；
② 选「检修操作」：🏥健康检查=只读体检最安全；🧹清理/💾备份=预设脚本；🔄服务重启=只重启服务；🔃重启服务器=重启整机（谨慎）；
③ 点「执行检修」，结果在下方"修复执行过程"卡片实时显示。
提示：脚本可点右上角 配置；Agent 离线时会提示先启动 Agent，不会报红错误堆栈。

 检修操作说明

                
            

            

                
各检修操作区别说明

                
                

                    

                        

                            
                        

                        

                            

                                

                                    
                                        

                                            操作
                                            说明
                                            影响范围
                                            使用场景
                                        

                                    
                                    
                                        

                                            🔄 服务重启脚本
                                            仅重启指定的应用服务（如nginx、mysql等）
                                            低影响 只影响单个服务
                                            服务异常、配置更新后生效
                                        

                                        

                                            🔃 重启服务器
                                            重启整个服务器操作系统
                                            高影响 所有服务中断
                                            系统更新、硬件问题、严重故障
                                        

                                    
                                

                            

                        

                    

                    
                    

                        

                            
                        

                        

                            

                                
点击检修面板右上角的  按钮打开脚本配置界面：

                                
    
                                    
  1. 使用内置命令：直接在"执行命令"文本框中编辑命令列表，每行一个命令
    2. 
                                    
  2. 使用自定义脚本：勾选"使用自定义脚本"，填入服务器上的脚本绝对路径，如 /opt/scripts/health_check.sh
    3. 
                                

                                

                                     
                                    注意：自定义脚本路径必须是目标服务器上的绝对路径，且需要有执行权限
                                

                            

                        

                    

                    
                    

                        

                            
                        

                        

                            

                                
Nginx

                                
systemctl restart nginx
# 或者
/usr/local/nginx/sbin/nginx -s reload

                                
                                
MySQL

                                
systemctl restart mysql
# 或者
service mysqld restart

                                
                                
Docker容器

                                
docker restart container_name

                                
                                
自定义脚本

                                
bash /opt/scripts/restart_app.sh
