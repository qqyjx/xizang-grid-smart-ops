# 西藏电网智能运维平台

基于**国网光明电力大模型（7B 多模态）+ 本地 ML/DL 双模式**的纯离线智能运维平台，服务于《2025 年基于国网光明电力大模型的智能运维场景的构建服务》项目，需求文档 18 项要点 **18/18 全覆盖**（智能检修 8/8 · 故障自愈 7/7 · 试点验证 3/3）。

> **关于 "7B 多模态"**：指国网内网大模型网关（"国网光明电力大模型"）后端部署的 **Qwen2-VL-7B-Instruct**（通义千问 2 视觉语言模型，约 70 亿参数），平台经网关的 OpenAI 兼容接口调用，具备监控截图/巡检照片的图文分析能力；网关不可达时自动回退本地 ML/DL 小模型。详见 [docs/AI模型架构.md](docs/AI模型架构.md)。

> 当前版本 v6.0 · 已在西藏电网多台生产 ECS 上部署试点验证

## 系统架构

```
监控服务器（平台端）                        被监控服务器 × N（Agent 端）
┌───────────────────────────┐            ┌────────────────────┐
│  platform/  端口 5001      │            │  agent/  端口 8089  │
│  Web 平台 + AI 分析引擎     │◄── HTTP ──►│  指标采集 + 巡检采集  │
│                           │   Token    │  修复命令执行        │
│  ┌─────────────────────┐  │   鉴权     │  故障注入演示        │
│  │ 本地小模型 ML + DL    │  │            └────────────────────┘
│  │ 7B 多模态大模型 API   │  │
│  │ 智能告警 + 根因分析    │  │
│  │ SOP 推荐 + 趋势预测   │  │
│  └─────────────────────┘  │
└───────────────────────────┘
            │
            ▼
   内网大模型网关（国网光明电力大模型 = Qwen2-VL-7B-Instruct，OpenAI 兼容接口）
```

- **纯离线**：现场无外网。安装包自带 Miniconda 与全部依赖；AI 分析默认走本地小模型，接入内网大模型网关后自动升级为 7B 多模态分析，两种模式无缝切换。
- **轻量 Agent**：被监控端仅部署一个轻量 HTTP 服务（psutil 采集 + 命令执行），最小依赖，支持 CentOS 7+ / Ubuntu 18.04+。
- **诚实数据原则**：Agent 离线时监控曲线不编造数据（诚实降级）；巡检只采集程序可真实获取的项目，应用层项目由人工录入补充，绝不虚构。

## 快速部署

```bash
# 平台端（监控服务器）
cd platform && chmod +x deploy.sh && ./deploy.sh     # Web: http://<监控服务器IP>:5001

# Agent 端（每台被监控服务器）
cd agent && chmod +x deploy.sh && ./deploy.sh        # 端口 8089
```

平台与 Agent 通过 `AGENT_TOKEN` 环境变量鉴权，两端必须一致。

## 安全说明

- 本仓库为交付展示版本，所有内网 IP、数据库口令、网关 API Key 均已用占位符替换，实际值由现场环境变量注入，不进入代码库。
- Agent 内置危险命令拦截：禁止 rm 系统关键目录、shutdown/reboot、mkfs、dd 写物理盘、fork 炸弹等。
